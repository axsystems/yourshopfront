# Apex Sites — Launch Checklist

Run this end-to-end on a Vercel staging deploy before promoting `redesign` to `master` and flipping production. Every item below must be checked or explicitly waived.

Sister docs: `REDESIGN-PLAN.md` (what we built), `REDESIGN-LOG.md` (the audit trail), `REDESIGN-REPORT.md` (final sign-off summary), `docs/phase-4-test-plan.md` (the manual Stripe runbook).

---

## 1. Repo state

- [ ] `master` and `redesign` are reconciled (PR merged or fast-forward).
- [ ] `git status` is clean. No untracked secrets in repo (run `git ls-files | xargs grep -l 'sk_live\|sk_test\|whsec_\|service_role'` — should return nothing).
- [ ] `pnpm install --frozen-lockfile` succeeds locally and on CI.
- [ ] `pnpm typecheck`, `pnpm lint`, `pnpm build`, `pnpm test:e2e` all pass on `master`.
- [ ] CI workflow (`.github/workflows/ci.yml`) is green on a fresh PR.

## 2. Vercel project

- [ ] Vercel project is connected to this repo.
- [ ] Production branch is set to `master` (or whichever is canonical).
- [ ] Build command: `pnpm build` (default for Next 16).
- [ ] Install command: `pnpm install --frozen-lockfile`.
- [ ] Node version: `20.x` (matches CI workflow).

## 3. Environment variables (Vercel → Production scope)

Reference: `.env.production.example` in repo root. Copy each line into Vercel and replace the `REPLACE` tokens with real values.

- [ ] `NEXT_PUBLIC_SITE_URL=https://apexsites.com`
- [ ] `STRIPE_SECRET_KEY` — **live mode**, not test (`sk_live_*`).
- [ ] `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` — live mode (`pk_live_*`).
- [ ] `STRIPE_WEBHOOK_SECRET` — generated by the live webhook endpoint after step 5 below.
- [ ] `STRIPE_PRICE_SUBSCRIPTION_SETUP` — live mode price ID (run `pnpm stripe:setup` with a live key).
- [ ] `STRIPE_PRICE_SUBSCRIPTION_MONTHLY` — live.
- [ ] `STRIPE_PRICE_ONETIME` — live.
- [ ] `STRIPE_PRICE_HOSTING_ADDON` — live.
- [ ] `SUPABASE_URL` — production project URL.
- [ ] `SUPABASE_SERVICE_ROLE_KEY` — production service-role key (server-only, no `NEXT_PUBLIC_` prefix).
- [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY` — production anon key.
- [ ] `RESEND_API_KEY` — production Resend key.
- [ ] `RESEND_FROM_EMAIL` — `Apex Sites <hello@apexsites.com>` (after DNS verification — see step 6).
- [ ] `CONTACT_INBOX_EMAIL` — `hello@apexsites.com`.
- [ ] `SLACK_WEBHOOK_URL` — production Slack webhook (optional but recommended).
- [ ] `NEXT_PUBLIC_PLAUSIBLE_DOMAIN=apexsites.com` (optional — leave empty to disable analytics entirely).

## 4. Supabase

- [ ] Production project exists in Supabase.
- [ ] Both migrations have been run in the SQL editor: `supabase/migrations/0001_initial.sql` and `0002_onboarding.sql`. Verify with:
  ```sql
  select count(*) from customers;     -- 0 expected on a fresh project
  select count(*) from sites;         -- 0 expected on a fresh project
  select column_name from information_schema.columns
    where table_name = 'sites' and column_name = 'onboarding_state';
  -- expected: 1 row
  ```
- [ ] RLS is enabled on `customers` and `sites`. Verify in Supabase dashboard → Authentication → Policies.
- [ ] No policies are defined for anon or authenticated roles (locked-by-default; service-role bypasses).

## 5. Stripe

- [ ] Stripe account is in **live mode**. Verify `dashboard.stripe.com/test/customers` and `dashboard.stripe.com/customers` show different sets.
- [ ] Run `pnpm stripe:setup` against the live key to idempotently create the 4 products. Capture the 4 emitted `STRIPE_PRICE_*=price_*` lines and paste into Vercel env vars (step 3).
- [ ] In Stripe Dashboard → Developers → Webhooks → Add endpoint:
  - URL: `https://apexsites.com/api/stripe/webhook`
  - Events to send: `checkout.session.completed`, `customer.subscription.deleted` (those are the only two the handler processes today).
  - Click "Reveal signing secret" → copy `whsec_*` → set `STRIPE_WEBHOOK_SECRET` in Vercel.
- [ ] **API version pin**: `src/lib/stripe.ts:15` pins to `2024-11-20.acacia`. The webhook endpoint Stripe creates will default to the account's current API version — verify it matches `2024-11-20.acacia` in the dashboard or events will be sent in a different schema and the handler will silently misread them.
- [ ] Run the manual end-to-end test plan in `docs/phase-4-test-plan.md` against the staging deploy. Sign off below:
  - [ ] Test 1 — Subscription tier (4242 card, $698 today). Site row, customer row, welcome email, Slack ping.
  - [ ] Test 2 — One-time tier without addon ($2,997). Site row, no subscription created.
  - [ ] Test 3 — One-time tier WITH addon ($3,026 today, $29/mo recurring).
  - [ ] Idempotency replay — `stripe events resend evt_*` does not create duplicate site rows.
  - [ ] Cancellation — `stripe subscriptions cancel sub_*` flips `sites.status` to `'cancelled'`, sends goodbye email.
  - [ ] Onboarding flow — content/assets/domain steps flip status to `'ready_to_build'`.

## 6. Resend (email)

- [ ] Production Resend account exists.
- [ ] **Domain verification**: in Resend → Domains → Add `apexsites.com`. Add the SPF + DKIM DNS records to Cloudflare (or your DNS host). Wait for both to validate (usually <10 min). Once green, switch `RESEND_FROM_EMAIL` from `onboarding@resend.dev` to `hello@apexsites.com`.
- [ ] Send test welcome email by running Stripe test 1 against the staging deploy and confirming delivery to a real inbox.
- [ ] Reply path works: replies to the welcome email land in `hello@apexsites.com` (forwards configured in your email host).

## 7. DNS

- [ ] `apexsites.com` A/AAAA records point to Vercel.
- [ ] `www.apexsites.com` redirects to `apexsites.com` (Vercel domain settings).
- [ ] Resend SPF + DKIM records present (see step 6).
- [ ] (If using Cloudflare for DNS) proxy is set to **DNS-only** for the apex; Vercel handles SSL.

## 8. SEO

- [ ] `robots.txt` resolves at `https://apexsites.com/robots.txt`.
- [ ] `sitemap.xml` resolves at `https://apexsites.com/sitemap.xml` and contains all 32 canonical URLs (1 home + 10 featured demos + 1 portfolio index + 14 portfolio details + 6 static pages).
- [ ] Submit sitemap to Google Search Console: `search.google.com/search-console` → add `https://apexsites.com` as a property → verify (DNS or HTML file) → Submit sitemap.
- [ ] Verify rich-results test on these URLs: `search.google.com/test/rich-results`
  - [ ] `https://apexsites.com/`
  - [ ] `https://apexsites.com/pricing`
  - [ ] `https://apexsites.com/portfolio`
  - [ ] `https://apexsites.com/portfolio/heritage-painters`
- [ ] OG image fallback `/og-default.png` resolves and renders.
- [ ] OG image dynamic route `/api/og/heritage-painters` returns image/png.
- [ ] Pre-warm the 24 dynamic OG images by curling each one once after deploy:
  ```
  for slug in ironside-plumbing greenwise-lawn bellhorn-movers ... ; do
    curl -sI "https://apexsites.com/api/og/$slug" >/dev/null
  done
  ```
  (Master brief noted this as a perf concern — `next/og` cold-fetches Fontsource on each cold edge invocation; CDN caches subsequent.)

  - [ ] `/` Performance ≥ 90
  - [ ] `/pricing` Performance ≥ 90
  - [ ] `/portfolio` Performance ≥ 85 (iframe gallery, brief §9 budget)
  - [ ] `/contact` Performance ≥ 90
  - [ ] `/about` Performance ≥ 90
  - [ ] `/demos/heritage-painters` Performance ≥ 90
- [ ] Accessibility score = 100 on every page above.
- [ ] If any page misses, the master brief §6.5 escalation applies (more aggressive lazy-load on `/portfolio`, etc.) — don't silently relax the budget.

## 10. Final smoke

- [ ] Click every link in the SiteHeader and SiteFooter — none should 404.
- [ ] Submit the contact form on `/contact` with a test email — confirm it lands in `hello@apexsites.com` and Slack pings.
- [ ] Click "Pick a style →" on the home, complete a checkout with `4242 4242 4242 4242` against the staging deploy — confirm the onboarding page renders and the 3-step checklist works.
- [ ] DemoSwitcher chips on `/demos/heritage-painters` navigate without flicker.
- [ ] PortfolioBanner on `/portfolio/heritage-painters` prev/next arrows work; "I want this look" links to checkout.
- [ ] Onboarding processing state polls every 5s when no `site` row exists yet (test by hitting `/onboarding?session_id=cs_test_invalid`).

## 11. Sign-off

- [ ] All boxes above checked or explicitly waived (with reason).
- [ ] Manual Stripe test plan signed off (`docs/phase-4-test-plan.md`).
- [ ] DNS propagation confirmed on at least 3 different networks.
- [ ] At least one human (you) has spent 10 minutes clicking around production as a real user.

When 1–11 are green, Apex Sites is live.
